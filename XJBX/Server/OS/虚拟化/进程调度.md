## 进程调度

首先定义一下turnaround time

```
T = 完成时间 - 事件到达时间
```

首先假设所有的任务都在同一时刻到达，且我们知道每一个任务的运行时间的话，

### FIFO

最简单的就是先进先出了，劣势比较明显，就是在负载大的任务排在队列前面的时候系统的周转事件是很长的。

### SJF最短任务优先

然后想到的就是去运行最短的任务，但是假如任务不是在同一时刻到达的话，还是会出现护航效应，例如最开始到达的是一个负载巨大的任务，之后的任务都比较轻巧，这个时候的周转时间还是没有缩小的

### 最短完成时间优先（STCF）

思考抢占式的调度程序，在SJF的基础上，剩下的工作和新工作谁先完成，然后调度该工作

### 半程总结

所以，假如知道每一个任务的运行时间的话，STCF会是一个很好的策略。但是响应时间也很关键，操作系统毕竟服务于人

```
响应时间=首次运行-到达时间
```

此时，STCF就不具有响应时间的优势了。

### 轮转

Roud-Robin是一种调度算法，其实就是时间片执行。这里需要考量的问题就是时间片的长度是多少才会不影响实际的执行性能，因为突然的时间片切换造成上下文切换造成的成本是影响性能的。另外，轮转还能结合IO，阻塞时，就没必要连同进程一起阻塞在哪里了，就直接切换就行了，然后io就绪的时候将该进程再假如到可切换的队列中
