## ThreadPool&&ProcessPool

最开始接触Pool是在游双的那本《高性能Linux后台编程》书中15章所写。最简单的网络编程下，创建子进程去完成并发服务器有很多缺点：

1. 耗时，动态创建线程或者进程是很昂贵的一件事，如果为了达到更高的并发。
2. 通常这种动态创建只能为一个客户服务，在线程的切换上天然处于弱势。
3. 动态的创建的新进程是当前进程的映像，必须仔细的管理文件描述符和其余的内存资源等

### 实现

线程池和进程池类似，这里一般由服务器预先创建出3-10个线程，这个数量应该和cpu的核数差不多，或者是两倍。

这里实现的线程池，使用一个工作队列，主线程往队列中插入任务，工作线程通过竞争来获取任务。
